Cloudbeds PMS Integration

This page provides instructions for signing up for devicethread and connecting it with Cloudbeds.

Pre-requisite: Ensure that you signed up for devicethread and have your property configured in devicethread.

Sign up for devicethread

Send an email to contact@devicethread.com, and we will assist you in signing up for devicethread and configuring your property.

Connect devicethread with Cloudbeds

The devicethread platform can be connected to Cloudbeds in one of the following ways.

Cloudbeds marketplace

Refer to this article to connect devicethread from Cloudbeds marketplace to your property

devicethread app

In the Property Dashboard, under Accounts go to “System Accounts” to connect and access your Cloudbeds account. This process typically involves authentication and authorization to establish a secure platform connection. Follow the on-screen instructions to complete the integration setup.

Enter your email and proceed to the next step in the integration process. Follow any additional prompts or steps required to securely connect devicethread to your Cloudbeds account.

Enter account details and sign in to Cloudbeds

Sign In screen

Enter the SMS verification code to authenticate your Cloudbeds account during the devicethread integration. This extra security step ensures authorized connections.

SMS Code verification screen

Allow all the necessary access permissions to enable devicethread to access Cloudbeds account details, especially for managing reservations. This ensures that devicethread has the required permissions to interact with Cloudbeds seamlessly and perform tasks related to reservations management.

Integration features

Below are the integration features from devicethread.

Lock code management

devicethread integration in Cloudbeds provides seamless management of lock codes for the rooms in the reservation. Following are the list of features:

  • 1. Unique code assigned to locks in the room(s) of the reservation
  • 2. For reservations with multiple rooms, devicethread assigns a unique lock code to each room. This ensures the security of all guests, as each room will have its own lock code.
  • 3. Lock code is delivered on a pre-configured interval before the check-in time ensuring no misuse of the code
  • 4. Lock code is updated in the reservation based on changes to the room
  • 5. Lock code is expired when the guest checks out or the guest fails to show up; and these actions are handled seamlessly with no additional effort from the front-desk executive

Email schedule with guest lock code

Cloudbeds allows configuration of email templates for every reservation status. You can use the lockcode property in the email template as required.

FAQs

Below are the integration features from devicethread.

Where do I find the guest lock code for the reservation in Cloudbeds?

Lock code will be visible in the reservation details page in Cloudbeds. Refer to the below image. You may have to click on the Show More button if the lock code section is collapsed, to view the details.

When does the lock code get delivered to the lock in a reservation?

The lock codes are delivered to Cloudbeds UI before the configured time ahead of the check-in. In case of walk-in, the code delivery is instantaneous.

Do I need to set the time zone of the property in devicethread?

Upon connecting devicethread with a Cloudbeds property account, devicethread synchronizes automatically the time zone of the property and uses it for displaying time on this zone in the devicethread app. You can view the time zone of the property in the property profile page under More -> Property Profile.

How is the lock code managed for multiple rooms in a reservation?

devicethread assigns a unique lock code for each room in a reservation to ensure security for all guests. These unique codes also work for accessing common areas within the property. The lock codes are securely managed and displayed within the devicethread.

How does devicethread handle the reservation status changes that happen in Cloudbeds?

devicethread manages the lifecycle of the lock codes based on the status of the reservation. Below are the different reservation status that devicethread handles:

Created

Lock code assigned for the reservation and the lockcode property displays the code in the reservation details in Cloudbeds UI. In case of property operating in full mode, the lock will be delivered to the lock(s) before the check in time.

In-house

Lock code delivery will be attempted when the reservation is assigned this status

Update

Possible updates to the reservation are:

  • 1. Check-in / Check-out time change. Lock code delivery and deletion in the lock(s) will be rescheduled

  • 2. Room change

    • If room(s) added to the reservation, the lock(s) corresponding the new room(s) will be added to the lock code delivery schedule

    • If room(s) removed from the reservation, the lock(s) corresponding the room(s) will be removed from the lock code delivery schedule

    • If the room is changed in the reservation, a new lock code will be generated and assigned to the reservation. The lock(s) corresponding to the changed rooms will be scheduled to receive the lock code

  • 3. Guest change. No changes will happen related to lock codes.

Do I need devicethread to manage reservations?

You can manage reservations under Access Management. From here, you can add and manage reservations. If the reservation has common areas as part of the collection, the lock code for common areas is currently visible only in devicethread app.

How does devicethread handle room changes in a reservation?

devicethread handles room changes in reservations as below:

  • If room(s) added to the reservation, the lock(s) corresponding the new room(s) will be added to the lock code delivery schedule

  • If room(s) removed from the reservation, the lock(s) corresponding the room(s) will be removed from the lock code delivery schedule

  • If the room is changed in the reservation, a new lock code will be generated and assigned to the reservation. The lock(s) corresponding to the changed rooms will be scheduled to receive the lock code
